# 常见问题

## Kaloscope 适合做什么

适合用于本地媒体库管理、自动追番、工作流驱动的资源站搜索与下载自动化。

## Kaloscope 和传统媒体管理工具有什么区别

最大的区别是：很多站点接入和自动化逻辑由工作流定义，而不是直接写死在程序代码里。这让系统更灵活，但也要求用户对工作流有基本理解。

## 可以直接用于生产环境吗

可以，但建议先完成以下检查：

- 使用独立用户运行
- 持久化工作区目录
- 对第三方工作流进行审查
- 结合反向代理、TLS 和鉴权方案部署

## Docker 镜像是否内置下载能力

镜像可选内置 aria2。设置 `ENABLE_ARIA2=true` 后，入口脚本会在容器内启动 aria2 RPC 服务。

## 第三方工作流是否安全

不一定。README 已明确提醒，第三方工作流可能包含任意代码或网络请求，导入前需要自行审查。

## 文档为什么有些章节还比较简略

当前文档优先完成了适合开源项目的结构骨架，并补充了已在 README、Dockerfile 中确认的信息。更具体的界面操作、配置示例和截图可以后续按模块逐步扩写。
